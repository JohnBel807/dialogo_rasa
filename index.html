Creando un Flujo Conversacional con Rasa

Una demostración interactiva de cómo los archivos de Rasa definen una conversación.

Los Componentes Clave de Rasa

Rasa utiliza varios archivos YAML para definir el comportamiento del bot. Los más importantes para un flujo simple son:

1. NLU (Natural Language Understanding)

Definición:

El archivo data/nlu.yml entrena al bot para que entienda lo que dice el usuario. Aquí definimos intenciones (lo que el usuario quiere hacer) y proporcionamos ejemplos de frases para cada una.

Ejemplo:

version: "3.1"

nlu:
- intent: greet
  examples: |
    - hola
    - buenos días
    - hey
    - qué tal

- intent: goodbye
  examples: |
    - adiós
    - hasta luego
    - nos vemos
    - chao

2. Dominio (Domain)

Definición:

El archivo domain.yml es el "universo" del bot. Define todas las intenciones que debe conocer, las respuestas (utterances) que puede dar, las entidades y las acciones que puede realizar.

Ejemplo:

version: "3.1"

intents:
  - greet
  - goodbye

responses:
  utter_greet:
  - text: "¡Hola! ¿En qué puedo ayudarte hoy?"
  
  utter_goodbye:
  - text: "¡Hasta luego! Que tengas un buen día."

3. Historias (Stories)

Definición:

El archivo data/stories.yml describe las conversaciones típicas. Mapea las intenciones del usuario a las acciones del bot, creando un "camino feliz" o flujo de la conversación.

Ejemplo:

version: "3.1"

stories:
- story: happy path
  steps:
  - intent: greet
  - action: utter_greet
  - intent: goodbye
  - action: utter_goodbye
